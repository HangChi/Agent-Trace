# 发布与版本策略

## 当前状态

Agent-Trace 仍处于内部迭代阶段，仓库没有 Git Tag 或正式稳定发行线。

- 根 `package.json` 的 `0.0.0` 表示私有工作区，不是用户发行版本。
- `apps/desktop/package.json` 的版本用于 Windows 桌面安装包文件名与升级识别。
- 在建立正式发布流程前，不承诺语义化版本兼容或长期支持周期。

## 版本来源

正式发布前应完成以下决策：

1. 明确发布物：桌面安装包、CLI/SDK npm 包或二者。
2. 选择唯一发行版本来源，避免根工作区与桌面包手工漂移。
3. 为发布提交创建带签名或受保护的 Git Tag。
4. 从 `CHANGELOG.md` 的 Unreleased 生成用户侧 Release Notes。

## 发布检查清单

- [ ] `pnpm verify` 在干净 checkout 中通过。
- [ ] Windows 解包目录和 NSIS 安装/卸载手工验收通过。
- [ ] 数据库升级、未来版本拒绝和数据保留测试通过。
- [ ] README、用户手册、API/OpenAPI、部署、安全与变更记录已同步。
- [ ] 安装包不包含开发数据库、WAL/SHM、环境文件或用户配置。
- [ ] 依赖许可证与第三方声明已审查。
- [ ] 已选择并由项目所有者批准代码许可证。
- [ ] 已配置安全报告渠道和发布制品校验值。

## 许可证状态

仓库当前没有许可证文件。没有明确许可证不等于开源授权；在项目所有者选择并批准 MIT、Apache-2.0、专有许可或其他条款前，不应把仓库描述为可公开再分发的软件。

许可证选择涉及所有权和法律授权，不能从技术代码推断。确定后应新增标准 `LICENSE` 文件，并在发布说明中记录生效范围与日期。
